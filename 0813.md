
```멘딕스에서 페이지를 만드는 방법을 단계별로 설명해드리겠습니다. 이 과정에서는 **Project Detail Page**를 예시로 사용하겠습니다. 이 페이지는 특정 프로젝트의 상세 정보를 표시하고, 관련 작업(Task)들을 관리할 수 있는 기능을 포함합니다.

### **1. 페이지 생성하기**

1. **모델러에서 앱 열기:**
   - 멘딕스 모델러를 열고 작업 중인 앱 프로젝트를 엽니다.

2. **페이지 추가:**
   - 프로젝트 탐색기(Project Explorer)에서 `Pages` 폴더를 우클릭합니다.
   - `Add Page...`를 선택하여 새 페이지를 추가합니다.

3. **페이지 템플릿 선택:**
   - `New Page` 창이 열리면, 사용할 템플릿을 선택해야 합니다.
   - `Data View` 템플릿을 선택합니다. 이 템플릿은 특정 데이터의 상세 정보를 표시하는 데 적합합니다.
   - 페이지 이름을 `ProjectDetailPage`로 설정하고, `OK`를 클릭하여 페이지를 만듭니다.

### **2. 데이터 소스 설정하기**

1. **Data View 위젯 추가:**
   - 새로 생성된 페이지에는 기본적으로 `Data View` 위젯이 포함되어 있습니다.
   - `Data View`의 데이터 소스를 설정하기 위해, `Data Source` 속성을 선택합니다.

2. **데이터 소스 연결:**
   - 데이터 소스를 `Context`로 설정하여, 이 페이지가 열릴 때 특정 `Project` 엔티티의 인스턴스를 받아오도록 합니다.
   - `Project` 엔티티를 선택한 후, 확인을 클릭합니다.

### **3. UI 요소 추가하기**

1. **프로젝트 정보 표시:**
   - `Data View` 내에서 프로젝트의 속성을 표시할 수 있는 UI 요소를 추가합니다.
   - `Name`, `Description`, `StartDate`, `EndDate`, `Status` 등의 속성을 표시할 텍스트 박스(`Text Box`), 날짜 피커(`Date Picker`), 드롭다운(`Dropdown`) 등을 추가합니다.
   - 각 속성을 `Project` 엔티티의 필드와 연결합니다.

2. **관련 작업 리스트 추가:**
   - `Data View` 아래에 `Data Grid` 위젯을 추가하여 프로젝트와 연결된 작업 리스트를 표시합니다.
   - `Data Grid`의 데이터 소스로 `Association`을 선택하고, `Project` 엔티티와 `Task` 엔티티 간의 연관 관계를 선택합니다.
   - `Task` 엔티티의 `Title`, `Status`, `DueDate`와 같은 필드를 `Data Grid`에 표시되도록 설정합니다.

3. **작업 추가/수정/삭제 버튼 추가:**
   - `Data Grid` 위젯 아래에 버튼을 추가하여 작업을 관리할 수 있는 기능을 제공합니다.
   - `Add Task` 버튼을 추가하여, 새 작업을 생성할 수 있는 폼으로 이동하도록 설정합니다.
   - `Edit Task` 버튼을 추가하여, 선택된 작업을 편집할 수 있도록 합니다.
   - `Delete Task` 버튼을 추가하여, 선택된 작업을 삭제할 수 있도록 합니다.

### **4. 마이크로플로우 연결하기**

1. **작업 추가 마이크로플로우 연결:**
   - `Add Task` 버튼을 클릭했을 때 실행될 마이크로플로우를 생성합니다.
   - 이 마이크로플로우는 새 작업을 생성하고, 현재 프로젝트와 연관을 설정한 후, 작업 생성 페이지로 이동하도록 설정합니다.

2. **작업 삭제 마이크로플로우 연결:**
   - `Delete Task` 버튼에 마이크로플로우를 연결하여, 선택된 작업을 삭제하고 데이터베이스를 업데이트하는 로직을 추가합니다.

### **5. 레이아웃 조정 및 스타일링**

1. **레이아웃 설정:**
   - 페이지의 레이아웃을 정리하여 사용자가 쉽게 정보를 파악할 수 있도록 합니다.
   - 그리드, 그룹박스(Group Box), 탭 컨트롤(Tabs) 등을 사용하여 정보를 카테고리별로 구분할 수 있습니다.

2. **스타일 설정:**
   - 페이지의 전반적인 디자인을 설정합니다.
   - 멘딕스에서 제공하는 기본 스타일을 적용하거나, 커스텀 CSS를 사용하여 페이지의 디자인을 사용자 정의할 수 있습니다.

### **6. 페이지 미리보기 및 테스트**

1. **페이지 미리보기:**
   - 상단 툴바에서 `Run` 버튼을 클릭하여 앱을 실행합니다.
   - 브라우저에서 앱이 실행되면, 생성한 `Project Detail Page`를 열어 제대로 동작하는지 확인합니다.

2. **테스트:**
   - 프로젝트 정보가 제대로 표시되는지, 관련 작업 리스트가 올바르게 표시되고 관리되는지 확인합니다.
   - 작업 추가, 수정, 삭제가 정상적으로 동작하는지 테스트합니다.

### **7. 배포**

1. **앱 배포:**
   - 모든 기능이 제대로 작동하는 것을 확인한 후, 앱을 배포합니다.
   - 멘딕스 클라우드나 자체 서버에 배포할 수 있습니다.
```
